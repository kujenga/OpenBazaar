Local node in the Kademlia network

This class represents a single local node in a Kademlia network; in other words, this class encapsulates an Entangled-using application's "presence" in a Kademlia network.

In Entangled, all interactions with the Kademlia network by a client application is performed via this class (or a subclass).

iterativeFindValue(self, key)

source code 

The Kademlia search operation (deterministic)

Call this to retrieve data from the DHT.

Parameters:
  • key (str) - the 160-bit key (i.e. the value ID) to search for
Returns: twisted.internet.defer.Deferred
This immediately returns a deferred object, which will return either one of two things:
  • If the value was found, it will return a Python dictionary containing the searched-for key (the key parameter passed to this method), and its associated value, in the format: <str>key: <str>data_value
  • If the value was not found, it will return a list of k "closest" contacts (kademlia.contact.Contact objects) to the specified key

_iterativeFind(self, key, startupShortlist=None, rpc='findNode')

source code 

The basic Kademlia iterative lookup operation (for nodes/values)

This builds a list of k "closest" contacts through iterative use of the "FIND_NODE" RPC, or if findValue is set to True, using the "FIND_VALUE" RPC, in which case the value (if found) may be returned instead of a list of contacts

Parameters:
  • key (str) - the 160-bit key (i.e. the node or value ID) to search for
  • startupShortlist (list) - A list of contacts to use as the starting shortlist for this search; this is normally only used when the node joins the network
  • rpc (str) - The name of the RPC to issue to remote nodes during the Kademlia lookup operation (e.g. this sets whether this algorithm should search for a data value (if rpc='findValue') or not. It can thus be used to perform other operations that piggy-back on the basic Kademlia lookup operation (Entangled's "delete" RPC, for instance).
Returns: twisted.internet.defer.Deferred
If findValue is True, the algorithm will stop as soon as a data value for key is found, and return a dictionary containing the key and the found value. Otherwise, it will return a list of the k closest nodes to the specified key

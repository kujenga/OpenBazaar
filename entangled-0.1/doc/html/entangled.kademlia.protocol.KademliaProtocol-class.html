sendRPC(self, contact, method, args, rawResponse=False)

source code 

Sends an RPC to the specified contact

Parameters:
  • contact (kademlia.contacts.Contact) - The contact (remote node) to send the RPC to
  • method (str) - The name of remote method to invoke
  • args (tuple) - A list of (non-keyword) arguments to pass to the remote method, in the correct order
  • rawResponse (bool) - If this is set to True, the caller of this RPC will receive a tuple containing the actual response message object and the originating address tuple as a result; in other words, it will not be interpreted by this class. Unless something special needs to be done with the metadata associated with the message, this should remain False.
Returns: twisted.internet.defer.Deferred
This immediately returns a deferred object, which will return the result of the RPC call, or raise the relevant exception if the remote node raised one. If rawResponse is set to True, however, it will always return the actual response message (which may be a ResponseMessage or an ErrorMessage).

_send(self, data, rpcID, address)

source code 

Transmit the specified data over UDP, breaking it up into several packets if necessary

If the data is spread over multiple UDP datagrams, the packets have the following structure:

   |           |     |      |      |        ||||||||||||   0x00   |
   |Transmision|Total number|Sequence number| RPC ID   |Header end|
   | type ID   | of packets |of this packet |          | indicator|
   | (1 byte)  | (2 bytes)  |  (2 bytes)    |(20 bytes)| (1 byte) |
   |           |     |      |      |        ||||||||||||          |

Note: The header used for breaking up large data segments will possibly be moved out of the KademliaProtocol class in the future, into something similar to a message translator/encoder class (see kademlia.msgformat and kademlia.encoding).
